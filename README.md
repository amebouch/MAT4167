# MAT4167
Bienvenue dans notre projet LaTeX. 

Alors, ce que j'ai réussi (on espère) à faire, c'est de permettre à notre projet d'exister dans le nuage pendant qu'on travaille localement sur notre ordinateur respectif.

J'utilise VScode pour éditer le code localement et je vous invite à faire la même chose.

Le concept c'est de périodiquement télécharger les updates et de upload nos propres updates quand on a travaillé dessus.
Donc, voici les étapes à suivre quand vous voulez apporter des changements (je me fie à ce que je fais sur VSCode, mais il se peut que ce soit différent si vous utilisez un autre logiciel):
Chaque fois que vous ouvrez VSCode, aller dans l'icone à gauche des trois cercles reliés par des fils et appuyez sur "sync changes".
Ensuite, allez travailler sur la section que vous voulez. IMPORTANT: on ne doit jamais être plusieurs à travailler sur la même section en même temps, sinon, il va y avoir des bugs - donc, quand on travaille sur une section, on peut le mentionner dans notre group chat pour que les autres n'y touchent pas. 
Une fois que vous avez travaillé sur une section, retourner dans l'icone à gauche. Vous allez voir un icone "Commit". En haut, il va y avoir un endroit pour écrire un message. Ce message sert à informer les autres des changements apportés. Par exemple, "Changements section 2". Donc, vous écrivez un message et ensuite, vous cliquez sur Commit.
Après avoir fait le Commit, il va apparaitre un bouton: "Sync changes". Cliquez sur cela. Ça va envoyer vos modifications sur GitHub. 

Il y a aussi quelques choses à setup pour que la communication entre Github et VScode se fasse bien.

(btw j'ai un mac so si vous êtes sur windows ou autre, c'est peut-être différent)

Moi j'ai installé (dans l'ordre) les choses suivantes:

-VSCode: https://code.visualstudio.com/download

-MacTex: https://www.tug.org/mactex/mactex-download.html

-LaTeX Workshop (VSCode Plugin): Nécéssite la configuration de certaines préférences.

-Homebrew: À travers terminal; la commande peut être trouvée sur https://brew.sh/

-Git: À travers terminal; la commande peut être trouvée sur https://git-scm.com/downloads/mac

-GitHub Pull Requests (VSCode Plugin): Nécéssite la connexion à un compte GitHub

Les instructions que j'ai suivies sont sur ce site: https://code.visualstudio.com/docs/sourcecontrol/github

Il est également nécessaire de configurer un nom d'utilisateur et une adresse de courriel sur Git dans le terminal. Les commandes peuvent être trouvées sur https://docs.github.com/en/get-started/git-basics/setting-your-username-in-git ainsi que sur https://docs.github.com/en/account-and-profile/setting-up-and-managing-your-personal-account-on-github/managing-email-preferences/setting-your-commit-email-address

Ce n'est peut-être pas super clair. Si vous avez déjà utilisé LaTeX et GitHub, ça devrait être plus simple.

Si on va de l'avant avec ce fonctionnement, le mieux c'est de travailler sur une seule section à la fois et d'en informer les autres afin de ne pas avoir de collisions. Mon idée c'est, par exemple, quelqu'un annonce dans le gc "je travaille sur la section X" puis il annonce "ok j'ai fini pour l'instant" quand les changements sont uploadés.

Laissez-moi savoir ce que vous pensez de tout ça!
